Git command

git ls-files -s 查看暂存区

初始化配置

git config –global user.name “ name”

git cofig –global user.email “email@e.e“

git config –list

C(新增)

在工作目录中新增文件

git status

git add ./

git commit -m “ msg”

D(删除&重命名)

git rm

git mv 旧文件 新文件名

git status

git commit -m “ msg”

R(查询)

git status

git diff 查看未暂存的修改

git diff –cache 查看未提交的暂存

git log –oneline 查看提交记录

分支

git log –oneline –decorate –graph –all 查看整个项目的分支图

git branch 查看分支列表

git branch -v 查看分支指向的最新的提交

git branch name 在当前提交对象上创建新的分支

git branch name commithash 在指定的提交对象上创建新的分支

git checkout name 切换分支

git branch -d name 删除空的分支 删除已经被合并的分支

git branch -D name 强制删除分支

git branch branchname 创建

git checkout branchname 切换

git checkout -b branchname 创建&切换

git branch -d branchname 删除

git branch -D branchname 强制删除

git merge branchname 合并分支

快进合并 不会产生冲突

​ 典型合并 可能会产生冲突

git branch 查看分支列表

git branch branchname commitHash 版本穿梭

git branch –merged 查看合并到当前分支的分支

​ 一旦出现在这个列表中 就应该删除

git branch –no-merged查看没有合并到当前分支的分支列表

​ 一旦出现在这个列表中就应该观察一下是否需要合并

注意点

在切换的时候一定要保证当前分支是干净的

​ 允许切换分支

​ 分支上所有的内容处于 已提交状态

​ （避免）分支上的内容是初始化创建 处于未跟踪状态

​ （避免）分支上的内容是初始化创建 第一次处于已暂存状态

​ 不允许切分支

​ 分支上所有的内容 处于已修改 或 第二次以后的已暂存状态

在分支上的工作做到一半时 如果切换分支 应该将现有工作储存起来

git stash 将分支上的工作推到一个栈中

分支切换 其他工作 完成后切回原分支

git stash apply 将栈顶工作内容还原但没有出栈

git stash drop 取出栈顶工作后 就应该出栈

git stash pop 等于以上两条

git stash list 查看存储

撤销

git checkout –filename 撤销工作目录的修改

git reset HEAD filename 撤销暂存区的修改

git commit –amend 撤销提交 修改注释

reset 三部曲

git reset –soft commithash 用commitHash内容重置HEAD

git reset [–mixed] commithash 重置HEAD 重置暂存区

git reset –hard commithash 重置HEAD暂存区工作目录

路径reset

所有的路径reset 都要省略第一步 第一步是重置HEAD 内容 head本质指向分支 分支本质是提交对象 提交对象 指向一个树对象 树对象可能指向多个git对象 git对象代表一个文件 head 可以代表一系列文件的状态

git reset [–mixed] commithash filename 用commithash中filename的内容重置暂存区

checkout深入理解

git checkout branchname 跟git reset –hard commithash像

都需要重置head 暂存区 工作目录

区别

​ checkout对工作目录是安全的 reset –hard 是强制覆盖

​ checkout动head时不会带着分支走而是切换

​ reset –hard时是带着分支走

checkout + 路径

​ git checkout commithash filename 重置暂存区 工作目录

​ git checkout –filename 重置工作目录

团队协作

  1. 初始化仓库
    在github上创建一个仓库
  2. 创建本地仓库
    git remote 别名 仓库地址
    git init 将源码复制进来
    修改用户名 邮箱
    git add
    git commit
  3. 推送本地仓库到远程仓库
    清理Windows凭据
    git push 别名 分支 推完后会生成远程跟踪分支
  4. 邀请项目成员 & 成员接受邀请
  5. 成员克隆远程仓库
    git clone 仓库地址 默认为远程仓库配的别名为orgin 默认主分支有对应的远程跟踪分支
  6. 成员修改源代码文件
    git add
    git commit
    git push 别名 分支 推完后会生成远程跟踪分支
  7. 更新修改
    git fetch 别名将修改同步到远程分支上
    git merge 远程跟踪分支 合并

本地分支

正常的数据推送和拉去

  1. 确保本地分支已经跟踪了远程跟踪分支
  2. 拉去数据 git pull
  3. 推送数据 git push

一个本地分支如何跟踪一个远程跟踪分支

  1. 当克隆时会自动生成一个master分支 已经跟踪了对应的远程跟踪分支
  2. 在新建其他分支时 可以指定想要跟踪的远程跟踪分支
    git checkout -b 本地分支名 远程跟踪分支名
    git checkout –track 远程跟踪分支名 用远程跟踪分支名作为本地分支名
  3. 将一个已经存在的本地分支 改成一个跟踪分支
    git branch -u 远程跟踪分支名

git branch -vv查看设置的所有跟踪分支

解决冲突

git add ./

git commit -m ‘’

git pull 拉去数据

git push 推送数据

删除远程分支

git push origin –delete serverfix 删除远程分支

git remote prune origin –dry -run 列出 仍在远程跟踪但是远程已被删除的无用分支

git remote prune origin 清楚上面命令列出来的远程跟踪分支

常用

git init 初始化仓库 git工作区

git status 查看文件的状态

git diff 查看哪些修改还没有暂存

git diff –staged 查看哪些修改暂存了但未提交

git log 查看提交历史记录

git log –oneline 查看提交历史记录 一行显示

git reflog 查看所有分支的操作的记录 包括已经删除的分支

git add ./ 将修改添加到暂存区

git rm 文件名 删除工作目录中对应的文件，再将修改添加到暂存区

git mv 原名 新文件名 将工作目录中的文件重命名再将修改添加到暂存区

git commit 将暂存区提交到版本库 并转到本次提交的注释页

git commit -a 将修改直接提交到版本库 并转到本次提交的注释页

git commit -a -m “注释” 将暂存区提交到版本库

git merge 分支名 把分支合并到当前分支 出现冲突后需要手动解决

git branch 显示分支

git branch 分支名 创建分支

git checkout 分支名 切换分支

git branch -D 分支名 强制删除分支

git branch -v 查看每一个分支的最后一次提交

git branch 分支名 提交对象的哈希 新建一个分支并且使分支指向对应的提交对象

git log –oneline –decorate –graph –all 查看项目分叉历史

git config –global alias.别名 命令 给命令起别名 （单个命令不需要加双引号）如：git config –global alias.st status git config –global alias.aaaa git log –oneline –decorate –graph –all

git checkout -b 分支名 新建一个分支并切换到新建的分支上

git stash 存储 将修改保存到一个栈上

git stash list 查看存储列表

git stash apply 重新应用存储的改动 如果不指定就默认用最新的 不会从栈中删除

git stash drop 存储名 从栈中移除存储

git stash pop 应用存储 并从栈中扔掉它

工作区

git checkout –filename 撤 工作目录中的修改 新增命令 效果相同git restore filename

暂存区

git reset HEAD filename 撤回暂存区中的暂存 git restore –staged 文件名

版本库

git commit –amend 重新修改注释 重新给用户一次机会注释

reset三部曲：

git reset –soft HEAD~ HEAD指针移到前一次提交 只动head(带着分支一起移动) 相当于放弃本次提交

git reset –soft hashcommit head指针移到 hashcommit

git reset –mixed HEAD~ 动HEAD 带着分支一起动 动了暂存区

git reset –hard HEAD~ 动HEAD 带着分支一起动 动了暂存区和工作区

git tag 列出标签

git tag -l “ ” 列出指定标签

git tag 标签 提交对象 添加标签 提交对象没有默认当前提交对象

git show 标签 查看标签

git tag -d 标签名 删除标签

git checkout 标签名 检出标签 git checkout -b 标签名 不创建分支的话 处于头部分离的状态
